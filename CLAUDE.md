## Build & Dev Commands

- `npm run dev` — start dev server (http://localhost:3000)
- `npm run build` — production build (Next.js 16 with Turbopack)
- `npm run start` — serve production build
- `npm run lint` — run ESLint (strict React 19 rules, no setState in effects)

## Architecture

Next.js 16 App Router + TypeScript + Tailwind CSS v4. A **365-day structured Japanese course** with day-by-day progression, interactive puzzles, and study history tracking.

### Course Data Engine (`src/data/course/`)

The course is built from a type system (`types.ts`) with three key types:

- **`DayDefinition`** — a single lesson day: title, summary, `ContentRef[]` for lesson content, `ExerciseSpec[]` for puzzles
- **`UnitDefinition`** — a group of related days (e.g., "Hiragana", "Grammar Basics")
- **`ContentRef`** — a pointer to content in the data files (e.g., `{ type: "hiragana", groups: ["vowels"] }`)

Days 1-30 are hand-written in `units/unit-01-hiragana.ts` and `units/unit-02-katakana.ts`. Days 31+ are generated by `generators.ts` which has functions `generateVocabUnit`, `generateKanjiUnit`, `generateGrammarUnit`, `generateMixedReviewUnit`. The course is assembled in `index.ts`.

**To add more course days**: expand the data files in `src/data/` (more kanji, vocab, grammar entries), then add generator calls in `src/data/course/index.ts` to create new units referencing that data.

### Content Data (`src/data/`)

Static TypeScript arrays. Each data file has its own typed interface:
- `hiragana.ts` / `katakana.ts` — `KanaChar[]` (46 chars each, grouped)
- `kanji.ts` — `KanjiChar[]` (20 N5 kanji with readings, examples, stroke counts)
- `vocabulary.ts` — `VocabWord[]` (60+ words across 8 categories)
- `grammar.ts` — `GrammarPoint[]` (10 N5 grammar points with examples)

### Exercise Engine (`src/lib/exercises.ts`)

`generateExercises(specs)` takes `ExerciseSpec[]` from a day definition and produces randomized `ExerciseQuestion[]`. It resolves `ContentRef` against data files to produce question/answer material. Five puzzle types:

| Type | Component | How it works |
|------|-----------|-------------|
| `multiple-choice` | `MultipleChoice.tsx` | Pick 1 of 4 options |
| `matching-pairs` | `MatchingPairs.tsx` | Click-to-pair two columns |
| `type-answer` | `TypeAnswer.tsx` | Type the answer (case-insensitive) |
| `fill-in-blank` | `FillInBlank.tsx` | Pick missing word from sentence |
| `drag-reorder` | `DragReorder.tsx` | Arrange word tiles (drag or tap-swap) |

All exercise components are in `src/components/exercises/`. `ExerciseRunner.tsx` orchestrates a question sequence, tracks score, and calls `onComplete`.

### Progression System (`src/lib/course-progress.ts`)

localStorage-based (`learnin-japan-course-v2` key). Key rules:
- Day N+1 unlocks when Day N completed with >= 60% score
- Streak tracks consecutive calendar days with activity
- `completeDay()` saves completion; `isDayUnlocked()` / `isDayCompleted()` check state

State is exposed to React via `useSyncExternalStore` in `src/lib/use-course-state.ts` — this avoids the `react-hooks/set-state-in-effect` ESLint rule. Always use `useCourseProgress()` hook instead of `useState` + `useEffect` for reading progress.

### Routes

| Route | Purpose |
|-------|---------|
| `/` | Home: stats, streak, "Continue" to current day |
| `/course` | Course map: units with day dots (green=done, red=current, grey=locked) |
| `/course/day/[n]` | Day page: lesson content + "Start Exercises" button |
| `/course/day/[n]/exercises` | Exercise session with score and pass/fail |
| `/history` | Activity calendar heatmap + completion log |

### Styling

Tailwind v4 with CSS custom properties in `globals.css`. Light/dark mode via `prefers-color-scheme`. Key tokens: `--accent` (red), `--success` (green), `--card`, `--card-border`, `--muted`. Uses Noto Sans JP font for Japanese text.

## Important Patterns

- **No setState in useEffect**: ESLint enforces this strictly. Use `useSyncExternalStore` for external state (localStorage). See `use-course-state.ts` for the pattern.
- **Content expansion**: The architecture supports 365 days but currently has ~95 days of content from existing data. To reach 365, expand `kanji.ts` (N4/N3 levels), `vocabulary.ts` (more categories), and `grammar.ts` (more points), then wire them in `course/index.ts`.
